Fase 1: Planejamento e Estruturação (Mês 1)
Sprint 01: Definição de escopo, criação de Roadmap, Kanban e Gráfico de Gantt.

Coleta de Dados: Pesquisa inicial sobre as fontes de dados de hardware e requisitos do sistema.

Fase 2: Infraestrutura e Prototipagem (Mês 2 - Atual)
Design de Interface: Finalização dos protótipos Web e Mobile (conforme o layout moderno de alta fidelidade).

Setup de Repositórios: Configuração do GitHub com acesso ao corpo docente.

Base de Dados: Implementação do esquema inicial em SQLite para suporte ao sistema de login.

Fase 3: Desenvolvimento do Core e Mobile (Mês 3 e 4)
Integração SQL: Finalização das queries e conexões de persistência de dados.

Desenvolvimento Mobile: Início da codificação em Flutter para o aplicativo PoupePC.

Fase 4: Pausa Acadêmica (Mês 5)
Férias de Julho: Interrupção programada conforme cronograma institucional.

Fase 5: Implementação de Back-End e Front-End (Mês 6 e 7)
Back-End (Python): Desenvolvimento da lógica de comparação e integração com fontes de dados.

Front-End (Web): Implementação da interface responsiva utilizando os protótipos como base.

Fase 6: Qualidade e Lançamento (Mês 8 e 9)
Testes de Integração: Verificação de fluxo entre Front, Back e Banco de Dados.

UX Adjustments: Ajustes finos focados na experiência do usuário e usabilidade.

Entrega Final: Apresentação do sistema funcional.
